## 한 줄 정의

> **workspace-hub** — 흩어져 있던 **TODO·캘린더·메모·작업공간 바로가기**를 한 곳에 모아주는 macOS 데스크톱 앱.

> 코드명은 `workspace-hub`. UI에 보이는 정식 표시 이름은 미정 (§ 열린 결정사항).

---

## 문제 정의 (Why)

흩어져 있어서 매번 새로 찾는다.

- **할 일 누락** — 어디에 적었는지 잊고, 마감일을 놓친다.
- **프로젝트 재진입 비용** — 같은 폴더·IDE·URL 묶음을 매번 다시 연다.
- **메모 표류** — 노트앱·슬랙 DM·메모지·README 초안 사이를 떠돈다.

→ 외부 서비스에 의존하지 않고 **로컬에서 빠르게** 동작하는 통합 도구가 필요하다.

---

## 사용자

- **주 사용자**: 현식 본인 — 개발자, macOS, 여러 프로젝트를 동시 진행.
- **부 사용자**: 동일 환경의 개발자/디자이너 — v0.1에서는 고려하지 않음.
- **사용 환경**: macOS 데스크톱 전용. 모바일/웹 클라이언트 없음.

> 단일 사용자 앱이다. 공유·협업·클라우드 동기화는 범위 밖.

## 기술 스택

이 프로젝트는 데스크톱 앱이므로 일반적인 web frontend/backend 구분이 아니라 **데스크톱 셸 + 로컬 CLI** 구조다.

### Desktop Shell

| 항목 | 값 |
|---|---|
| 프레임워크 | Tauri v2 |
| 셸 언어 | Rust |
| 프론트엔드 | React + TypeScript |
| UI 라이브러리 | 자체 CSS 변수 + 컴포넌트 (Notion 스타일 토큰 직접 적용, 외부 UI 키트 없음) |
| 빌드 / 패키지 매니저 | pnpm |

### Core 라이브러리 (도메인 로직·DB 접근의 단일 진실 원천)

| 항목 | 값 |
|---|---|
| crate | `workspace-hub-core` (workspace member) |
| 언어 | Rust (Tauri와 동일 언어 → workspace 공유) |
| DB 드라이버 | `rusqlite` + `bundled` feature (정적 링크, CGO 불필요) |
| 마이그레이션 | `core/migrations/V###__*.sql` 정적 임베드 + `schema_version` 테이블 |
| 에러 처리 | `thiserror` |
| 데이터 직렬화 | `serde` / `serde_json` |

### CLI (Claude Code 스킬용 외부 인터페이스)

| 항목 | 값 |
|---|---|
| 바이너리 | `workspace-hub` (`cli/` crate) |
| CLI 프레임워크 | `clap` v4 |
| 역할 | `core` 를 감싸 JSON/사람 친화 출력을 stdout 으로 emit. Tauri 셸은 사용하지 않는다(ADR-0012). |

### 데이터

| 항목 | 값 |
|---|---|
| 데이터베이스 | SQLite (`core` 만 접근) |
| 데이터 위치 | `~/.workspace-hub/` (사용자 홈 디렉터리 직속 dotfile — Tauri 앱·CLI 가 모두 동일 위치 참조) |
| 외부 서비스 | **없음** — 모든 네트워크 호출 금지 |

### 호출 흐름

```
[React UI] ──IPC──► [Tauri Rust Shell] ──fn call──► [core lib] ──► [SQLite]
                                                         ▲
                          [Claude Code skills] ──spawn──►[CLI bin]┘
```

- Tauri 셸은 `core` 를 cargo workspace 의존으로 직접 호출한다(in-process). 앱 시작 시 `db::open()` 으로 단일 `Connection` 을 열고 `Mutex<Connection>` Tauri State 로 공유.
- Claude Code 스킬은 `workspace-hub` CLI 바이너리를 Bash 로 spawn 한다. CLI 는 같은 `core` 를 호출하므로 도메인 동작은 두 경로가 동일.
- CLI 입출력은 **기본 JSON, `--human` 플래그로 표 형식**. stdout=데이터, stderr=진단/로그.

> 위 결정의 배경은 [`ADR.md`](./ADR.md) 참조. 코드 디렉터리 구조는 [`SOURCE_MAP.md`](./SOURCE_MAP.md).

## 범위 (Scope)

### v0.1 — MVP에 포함되는 것

**TODO**
- 할 일 추가/수정/완료/삭제 (제목·설명·마감일·우선순위 0–3)
- 워크스페이스에 귀속(선택)
- 완료 항목 별도 보기/숨기기, 마감 지난 미완료 항목 시각 강조
- 마감일 있는 TODO는 캘린더에 자동 표시

**캘린더**
- 월/주/일 뷰
- TODO 마감일 + 자체 이벤트(제목·시작/종료·종일 여부·설명·장소·색상) 통합 표시
- 외부 캘린더(Google, iCloud) 연동 **없음**

**메모**
- 추가/수정/삭제, 제목+본문(서식 있는 텍스트)
- 워크스페이스 귀속(선택), 검색(제목+본문), 고정(pin)

**워크스페이스**
- 작업 그룹(예: Personal / Work / Side Project A)
- 추가/수정/삭제/순서 변경, 이름·색상·아이콘
- 묶을 수 있는 항목: 디렉토리 바로가기(Finder) / 앱 바로가기 / URL 바로가기
- 워크스페이스 선택 시 해당 TODO/메모만 필터링

**공통**
- 라이트(흰색 캔버스) 단일 테마 — Notion 톤 (ADR-0009)
- 사이드바로 영역 전환 (TODO / 캘린더 / 메모 / 워크스페이스)
- 모든 데이터 로컬 저장 — 외부 네트워크 호출 없음
- 재실행해도 데이터 유지

## 사용자 시나리오

### A. 아침에 앱을 열었을 때
1. 사이드바 기본 선택은 **TODO**.
2. 상단에 오늘 마감, 그 아래에 **overdue**(빨강)가 보인다.
3. "Work" 워크스페이스 선택 → 묶인 디렉토리/앱/URL 카드가 보인다.
4. 카드 클릭 → Finder/앱/브라우저로 열림.

### B. 회의 중 메모 적기
1. **Memos** → "새 메모" → 제목+본문 입력 → 즉시 저장.
2. "회의록" 키워드로 검색해서 다시 찾음. 자주 보는 메모는 핀 고정.

> v0.1에는 글로벌 단축키 없음 — Cmd+Space 같은 빠른 진입은 v0.3에서 검토.

### C. 새 프로젝트 시작
1. **Workspaces** → 새 워크스페이스 "Side Project A" 생성, 색상 지정.
2. 디렉토리(`~/projects/side-a`), 앱(IntelliJ), URL(Jira) 추가.
3. 이후 이 워크스페이스에 묶이는 TODO/메모만 사이드바 필터로 분리해 본다.

### D. 마감일 임박 확인
1. **Calendar** → 이번 달 뷰에 TODO 마감일과 이벤트가 함께 표시.
2. 특정 날짜 클릭 → 그날의 TODO+이벤트 목록.
3. 캘린더에서 직접 새 이벤트 추가.

## 성공 기준

### 정성적
- 출근 후 "오늘 뭐 해야 하지" 확인하러 다른 앱을 안 연다.
- 프로젝트 전환 시 "그 폴더 어디였더라" 검색을 안 한다.
- 회의 중 메모 위치를 다시 찾기 위해 노트앱을 안 뒤진다.
- 결과적으로 **하루 5회 이상 자연스럽게 열게 되는** 앱이 된다.

### 정량적 (출시 후 1개월 자가 평가)
- TODO 누적 등록 100건 이상
- 활성 워크스페이스 3개 이상
- 메모 30건 이상 등록 & 그중 1건 이상 재방문
- 앱 충돌/데이터 손실 0건

## 비기능 요구사항 (UX 관점)

- **즉시성** : 앱 실행 후 3초 안에 메인 화면. 입력 끊김 없음.
- **신뢰성** : 입력하는 순간 저장. "저장" 버튼으로 잃어버리는 경험 금지.
- **로컬-우선** : 인터넷 없어도 모든 기능 동작.
- **프라이버시** : 어떤 데이터도 외부로 나가지 않는다.
- **시각적 안정감** : 흰색 캔버스 + 1px hairline 보더의 sober-editorial 톤. 알림창/뱃지가 시끄럽지 않다.
- **macOS 네이티브 느낌** : 단축키·창 컨트롤은 macOS 관습을 따른다. 테마는 시스템 설정과 분리된 단일 라이트.

## 비고 / 의도적 제약

> 코드 결정의 배경이 되는 "하지 않기로 한 것".

- **macOS 데스크톱 전용** — 다른 OS / 웹 / 모바일은 만들지 않는다.
- **단일 사용자** — 인증·다중 계정·user_id 컬럼 없음.
- **로컬 우선** — 외부 네트워크 호출은 금지. 동기화·클라우드는 v0.5 이후.
- **외부 캘린더 미연동** — Google/iCloud 연동은 의도적으로 빼고, 앱 내부 캘린더만.
- **i18n 없음** — 한국어 + 영어 혼용 UI를 허용. 본격 다국어는 보류.
- **SQLite는 CLI 전용** — Tauri는 직접 DB에 접근하지 않는다. 락 충돌·이중 진실 원천을 피하기 위함.
- **배포는 로컬 빌드만** — DMG·App Store·외부 다운로드 없음. 본인이 저장소를 clone해 직접 빌드해 쓴다. 따라서 코드 서명·노타라이즈도 v0.1에서는 생략.
